## 🛠️ Getting Started

Clone the repo and use the Python engines directly, or use the Docker decoder for a hardened offline restore path. No license required for anything.

### One-Command Installation
```bash
git clone https://github.com/Parad0x-Labs/liquefy.git
cd liquefy
chmod +x ./liquefy
```

### Usage (Linux/macOS)
```bash
# Decompress a production archive
./liquefy decompress archive.null restored.log

# Verify bit-perfect integrity
./liquefy verify archive.null
```

### Windows (PowerShell)
```powershell
.\liquefy decompress archive.null restored.log
```

> **Note:** The `./liquefy` wrapper is a Bash script. On Windows use WSL, Git Bash, or Docker Desktop with a Linux container.

## 📂 Offline Decompression (Public, Decode-Only)

Liquefy guarantees that **your data is never hostage**. Anyone can **decompress and verify** Liquefy archives locally using the public **decode-only** appliance.

The `./liquefy` wrapper runs a **pinned decoder image** published by Parad0x Labs.

### Notes
- **Capabilities:** Decompression + integrity verification only (no compression).
- **Offline by default:** The decoder runs with `--network=none` and a read-only root filesystem. Your data never leaves the machine.
- **First-run / restricted networks:** The wrapper may need **one-time registry access** to pull the pinned decoder image. If registry access is blocked, preload the image (airgap-friendly) using the steps below.

### Airgapped / Restricted Network Quickstart

> Replace `<DECODER_IMAGE:PINNED_TAG>` with the **exact** image reference used by `./liquefy`.

**On a machine with registry access (staging/build box):**
- Pull + export the pinned decoder image:
  - `docker pull <DECODER_IMAGE:PINNED_TAG>`
  - `docker save -o liquefy-decoder-image.tar <DECODER_IMAGE:PINNED_TAG>`

Transfer `liquefy-decoder-image.tar` to the airgapped host (USB / secure artifact store).

**On the airgapped host:**
- Import the image locally:
  - `docker load -i liquefy-decoder-image.tar`
- Run as usual (still offline / `--network=none`):
  - `./liquefy verify <archive>.null`
  - `./liquefy decompress <archive>.null <output>`

**Tip:** If your org uses an internal registry mirror, push the pinned image there and pull from the mirror instead of a public registry.

## 🐳 Docker Decoder Policy

The $NULL Sovereign SDK is built for **enterprise-grade reliability**. To ensure secure conduction and data sovereignty, the following policies are enforced:

1.  **Maintenance & Compatibility:** This build is optimized for archives generated by v3.1 cores. While decompression is never paywalled, newer archive formats may require the latest signed build of the SDK.
2.  **The Docker appliance detects unauthorized runtime hooks and fails closed.**
3.  **Data Sovereignty:** All operations happen locally. No data is ever transmitted back to Parad0x Labs.

## 🔧 Engine Source (MIT)
The compression engine source is in the `engines/` directory:
- `engines/json_codec/` — JSON / structured log columnar compression
- `engines/nginx_codec/` — Nginx access log optimised codec
- `engines/apache_codec/` — Apache access log codec
- `engines/syslog_codec/` — Syslog RFC 3164/5424 codec
- `engines/k8s_codec/` — Kubernetes JSON stream codec
- `engines/sql_codec/` — PostgreSQL / SQL query log codec
- `engines/aws_codec/` — AWS CloudTrail / VPC Flow codec
- `engines/universal_codec/` — Fallback entropy codec
- `engines/orchestrator.py` — Hyper-orchestrator (auto-routes to best codec)
- `tools/benchmark.py` — Head-to-head benchmark vs zstd
